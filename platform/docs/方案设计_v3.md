# 百草园周报填报平台 - 方案设计 v3

> 版本: v3 (基于 5 轮讨论)
> 日期: 2026-06-05
> 形态: 设计方案 + HTML/MockJS/TailwindCSS 可跑通原型

---

## 0. 范围与定位

**目标**：把"每周手动拼 12 个 HTML 页 + 出 PDF"的工作，压缩到**"AI 出报告 + 看异常 + 一键导出"**。

**当前阶段**：设计方案 + 静态原型。**不接真实 LLM**，所有 AI 行为用 MockJS 模拟，但接口、prompt、解析逻辑按真实设计走。

**技术栈**：纯前端，HTML + TailwindCSS (CDN) + MockJS (CDN) + 原生 ES 模块。**不引入 React**——保证原型 0 构建，浏览器直接打开 index.html 就能跑。

## 1. 组织模型

```
清尚 (填报方)                                中建三局 (接收方)
┌────────────────────────┐                ┌────────────────────┐
│ 项目 A 百草园            │  ── 提交/审批 ──>  │ 总包周报汇总         │
│   ├ 项目档案 (花名册/里程碑)  │                │ (本期不做拼合)        │
│   ├ 周报实例 (W18, W19…)  │                └────────────────────┘
│   └ AI 配置 (Prompt 模板)  │
└────────────────────────┘
```

**v1 角色**：
- `填报人` — 清尚项目上，负责录入+审异常
- `复核人` — 清尚项目负责人，最终确认
- `阅读人` — 中建三局对接人，只读+批注

## 2. 章节与填报强度

| 页 | 章节 | 填报强度 | 录入入口 | AI 自动化 |
|---|---|---|---|---|
| 01 | 封面 | 🟢 自动 | 元信息带出 | 无 |
| 02 | 目录 | 🟢 自动 | 模板生成 | 无 |
| 03 | 组织架构 | 🟢 低 | 花名册+本周到岗 | 比对上周 |
| 0301 | 里程碑 | 🟢 低 (季度动) | 项目档案 | 固化 |
| **04** | **上周工作完成** | 🔴 **高** | **语音/粘贴/录入/附件** | **抽取+异常** |
| 05 | 上周工作现场 | 🟡 中 | 拍照+选图 | AI 分组 |
| **06** | **人员统计** | 🟡 **中** | **花名册+签到** | **自动汇总** |
| 07 | ECC 销项 | 🟡 中 | 截图+Excel | OCR+算关闭率 |
| 08 | 图纸深化 | 🟡 中 | 语音/粘贴/录入 | 抽取 |
| **09** | **下周周计划** | 🔴 **高** | **语音/粘贴/录入** | **排甘特** |
| 10 | 施工段划分 | 🟡 中 (图每周更新) | 上传+标注 | 无 |
| 11 | 施工段计划 | 🟡 中 | 录入+续排 | 续排+异常 |
| 12 | 协调事宜 | 🟢 低 (常空) | 录入 | 无 |

**LLM 4 个主战场**：04、06、07、08、09、11（实际 6 个，11 也用得上）

## 3. 多入口输入设计

每个 AI 章节提供 **4 种入口** 之一或多种：

| 入口 | 适用场景 | 实际接口 (v1 用 Mock 模拟) |
|---|---|---|
| 🎤 **语音** | 在工地现场，腾不出手打字 | Web Speech API → 文本 → LLM 抽取 |
| 📋 **粘贴** | 微信群聊天、长文本 | 直接文本 → LLM 抽取 |
| ✍️ **录入** | 不喜欢 AI，想自己填 | 表格内联编辑 |
| 📎 **附件** | PDF/Word/Excel/图片 | 文件上传 → 解析 → LLM 抽取 |

**UI 模式**：每个章节顶部一条"快捷录入区"，4 个 Tab 切换，AI 抽取后展示**待确认的草稿列表**，人点"接受"才入库。

## 4. AI 介入：自动驾驶 + 安全员

```
AI 默认全量出报告 ──→ 异常检测 ──→ 异常推人 ──→ 人工修 ──→ 导出
```

**异常分级**：

| 等级 | 颜色 | 触发 | 阻塞导出 |
|---|---|---|---|
| 🔴 红 | `bg-red-50 border-red-300` | ECC 关闭率 < 70% / 关键节点延误 / 关键岗位空缺 | 是 |
| 🟡 黄 | `bg-amber-50 border-amber-300` | 上周"进行中"未续排 / 工种人数 0 但工序在排 / 劳动力偏离 >30% | 否 |
| 🔵 蓝 | `bg-sky-50 border-sky-300` | 字段缺失 / 截止日期空 / 责任人未填 | 否 |

## 6. 异常检测规则 (v1 规则引擎)

```js
// platform/js/rules.js (Mock)
const RULES = [
  {
    id: "ecc-low-close-rate",
    level: "red",
    check: (r) => {
      const rate = r.ecc_summary.closed / r.ecc_summary.total;
      return rate < 0.7;
    },
    msg: (r) => `ECC 关闭率 ${(r.ecc_summary.closed/r.ecc_summary.total*100).toFixed(2)}%，低于 70% 基线`
  },
  {
    id: "carry-over-untouched",
    level: "yellow",
    check: (r) => {
      // 上周 progress<100% 且 owner/任务文本 本周未出现的任务
      // v1 简化：仅基于"上周"快照
      return r.ai_review?.evidence?.continued_from_last_week?.length > 0;
    },
    msg: (r) => `上周 ${r.ai_review.evidence.continued_from_last_week.length} 项进行中，本周已续排`
  },
  {
    id: "empty-coordination",
    level: "blue",
    check: (r) => r.coordination.every(c => !c.issue),
    msg: () => `协调事宜表为空，确认无待协调事项？`
  },
  {
    id: "missing-owner",
    level: "blue",
    check: (r) => r.work_done.some(t => !t.owner),
    msg: (r) => `上周工作完成中有 ${r.work_done.filter(t=>!t.owner).length} 条缺责任人`
  },
  {
    id: "labor-spike",
    level: "yellow",
    check: (r) => {
      const total = r.labor_stats.find(l => l.is_total);
      const delta = (total.next_week - total.this_week) / total.this_week;
      return Math.abs(delta) > 0.3;
    },
    msg: (r) => {
      const t = r.labor_stats.find(l => l.is_total);
      const d = ((t.next_week - t.this_week)/t.this_week*100).toFixed(1);
      return `总人数下周较本周 ${d>0?'+':''}${d}%，超 30% 阈值`;
    }
  }
];
```

## 7. 系统 UI 风格：中性灰+蓝

**色板**：

| 角色 | 值 | 用途 |
|---|---|---|
| `bg-app` | `#f8fafc` (slate-50) | 背景 |
| `bg-panel` | `#ffffff` | 卡片 |
| `border-subtle` | `#e2e8f0` (slate-200) | 卡片描边 |
| `text-primary` | `#0f172a` (slate-900) | 标题 |
| `text-secondary` | `#475569` (slate-600) | 正文 |
| `text-muted` | `#94a3b8` (slate-400) | 辅助 |
| `accent` | `#2563eb` (blue-600) | 主操作 |
| `accent-soft` | `#dbeafe` (blue-100) | 高亮底色 |
| `success` | `#10b981` | 通过 |
| `warning` | `#f59e0b` | 提示 |
| `danger` | `#ef4444` | 异常 |

**版式**：左侧固定 220px 侧边栏（项目+周期切换 + 12 章节导航），右侧主区（章节表单/预览）。

## 8. 部署形态

- **测试**：内网，**纯静态**，Nginx/Apache 直接 serve 即可，**无需后端**
- **正式**：私有云（VPC），未来加 FastAPI 后端 + 对象存储（图片/PDF 附件）
- **数据**：v1 用 localStorage 模拟持久化；v2 切换为 PostgreSQL JSONB
